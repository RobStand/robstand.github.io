Quiddity

A browser-based IDEF5 ontology modelling tool

What is Quiddity?

Quiddity is a visual modelling tool for building IDEF5 ontology schematics: diagrams that formally describe the kinds of things that exist in a domain, the relationships between them, and how those things change over time.

The application runs entirely in your browser as a single HTML file. No installation, no server, no account required. Just open it and start drawing.

New to IDEF5? See About IDEF5 at the bottom of this page for a brief introduction to the notation.

Interface overview

The Quiddity window has four main areas:

Area Location Purpose
Toolbar Top bar (dark) File operations, undo/redo, zoom, and loading examples
Toolbox Left panel Palette of all IDEF5 symbols. Drag onto the canvas to place them.
Canvas Centre The drawing surface where your ontology diagram lives
Properties panel Right panel Editable fields for the currently selected element (label, notes, color, etc.)

Quick start

The fastest way to explore Quiddity is to load one of the built-in examples:

  1. Click Examples ▾ in the toolbar and choose Ballpoint Pen Components.
  2. Browse the diagram. Hover over symbols in the toolbox; tooltips explain each one.
  3. Click any symbol to select it; its properties appear in the right panel.
  4. Double-click a node to edit its label directly on the canvas.
  5. When you are ready to build your own model, click New in the toolbar.
Tip: Try the Water Phase Transitions and Vehicle Classification examples to see different IDEF5 diagram styles.

Navigating the canvas

Pan

Hold Space and drag with the mouse to pan. Alternatively, drag on any empty area of the canvas.

Zoom

  • Scroll the mouse wheel to zoom in and out, centred on the cursor.
  • Click + / − in the toolbar for step zooms.
  • Click Fit to zoom the diagram to fill the window.

Adding nodes

Every IDEF5 symbol starts in the toolbox on the left.

  1. Find the symbol you want in the toolbox.
  2. Drag it onto the canvas and release. The node is placed where you drop it.
  3. Nodes snap to a 10-pixel grid automatically.
Note: Some toolbox items are connector tools, not standalone nodes. Dragging them from the toolbox starts a two-click connection workflow instead of placing a node. These are: Second-Order Relation, Alt. 2-Place Relation, State Trans. (Weak), State Trans. (Strong), and all three Connect symbols.

Drawing connections

Connector tools (two-click workflow)

There are two ways to start a connection:

  • Click a connector item in the toolbox to enter connection mode, then click the source node.
  • Drag a connector item from the toolbox directly onto the source node to enter connection mode with the source already set.

Either way, finish by clicking the target node. A dashed preview line tracks your cursor while the target is being picked.

Press Esc at any point to cancel the operation.

First-order relation connections

Place a first-order relation rectangle on the canvas (drag 1st-Order Rel. from the toolbox), then connect its edges to kind or individual nodes by drawing connector lines from node to node.

State transition tip

When you click on the midpoint circle of an existing state transition node while drawing a connection, Quiddity automatically splits the transition and inserts a proper standalone transition node at that position.

Editing elements

Label

  • Double-click any node to open an inline text editor directly on the canvas.
  • Press Enter or click elsewhere to confirm; press Esc to discard.
  • Labels can also be changed in the Properties panel on the right.

Notes

Every node has a free-text Notes field in the Properties panel. Use it to record definitions, source references, or modelling decisions. Notes are saved with the diagram but are not displayed on the canvas.

Color (kinds and individuals)

With a kind or individual selected, a color swatch grid appears in the Properties panel. Click a color to fill the node with that color at 50% opacity. The default swatch (white with a cross) resets to no color.

Resizing

Select a resizable node (relation rectangle, process, referent, or container). Blue corner handles appear. Drag any handle to resize the node.

Referent fields

A Referent node exposes two additional fields in the Properties panel: Reference ID and Method Name. The main label field sets the concept label shown in the bottom half of the node.

Key entries

A Key node shows an Entries section in the Properties panel. Use the symbol-type selector and label input to add entries. Remove any entry with the red × button. The key on the canvas updates live.

Selecting & moving

Single selection

Click any node or edge to select it. The element highlights in blue and its properties appear on the right.

Multi-selection

  • Hold Shift and click additional elements to add them to the selection.
  • Click-drag on empty canvas space to draw a rubber-band selection rectangle around multiple elements.

Moving

Drag any selected node to move it. If multiple nodes are selected, the whole group moves together. Nodes snap to the 10px grid during movement.

Deselect

Click on empty canvas space to deselect everything.

Context menu

Right-click any node or edge to open the context menu:

ActionDescription
DeleteRemove the element (and any edges connected to it) from the diagram
DuplicateCreate an offset copy of the element
Bring to FrontMove the element on top of overlapping elements
Send to BackMove the element behind overlapping elements

Kinds & Individuals

Kind
A natural category of things that share essential, defining properties. Depicted as a circle. Kinds form the ontological backbone of an IDEF5 model: the classes whose nature the modeller is capturing.
Individual
A specific, named instance of a kind. Depicted as a circle with a filled dot at the bottom. The dot distinguishes it from a kind. In the Properties panel, any kind can be toggled to an individual without deleting it.

Relations

1st-Order Relation
A relationship whose arguments are kinds or individuals. Depicted as a rounded rectangle. Draw edges from the rectangle to the participating kinds. Examples: "contains", "is made of", "precedes".
2nd-Order Relation
A relation that takes another relation as one of its arguments (a "relation about a relation"). Depicted as a line with a filled triangle at the source end. The triangle tip connects to the target relation line. This is a connector tool: drag it from the toolbox, then click source and target.
Alt. 2-Place Relation
An alternative notation for a 2-place first-order relation. The relation name sits directly on the connecting line rather than in a separate rectangle, which is useful for compact diagrams. This is also a connector tool.

Subkind, instance-of, and part-of edges

These three fundamental IDEF5 relationships are expressed as edge types on connections between nodes:

Edge typeMeaningVisual style
Subkind-of The source kind is a specialisation of the target kind. Every instance of the subkind is necessarily an instance of the superkind. Dashed line, open triangular arrowhead pointing to the parent kind
Instance-of The source individual is a member of the target kind. Dashed line, filled arrowhead pointing to the kind
Part-of Mereological (compositional) relationship: the source is a part or constituent of the target. Solid line, arrowhead pointing to the whole

Process

Process
An activity that transforms inputs into outputs. Depicted as a rectangle with a bottom shelf and a central divider on that shelf. The label goes in the upper area. Connect processes to state nodes and junctions using the Connect symbols.

State Transitions

State Transition (Weak)
A possible but non-guaranteed state change. The open circle at the midpoint is the transition marker. Connect incoming arcs (the "before" states) and outgoing arcs (the "after" states) to the circle. Single arrowhead at the right end.
State Transition (Strong)
A necessary and irreversible state change. Identical in structure to the weak transition but with a double arrowhead at the right end. Connect incoming and outgoing arcs to the midpoint circle.
Instantaneous Transition
A state change that occurs instantaneously with no intermediate duration. Depicted as a triangle (Δ). Used when the time of the transition is negligible or undefined.

Junctions

Junctions control flow logic in process and state diagrams. They express how multiple paths combine or diverge.

Junction (XOR)
Exactly one incoming path must be satisfied. Represents mutually exclusive alternatives: one and only one branch is taken.
Junction (OR)
At least one incoming path must be satisfied. Represents inclusive alternatives: one or more branches may be taken.
Junction (AND)
All incoming paths must be satisfied. Represents concurrent or joint conditions: all branches must complete before flow continues.

Connecting Symbols

Connecting symbols join processes, junctions, and state nodes in flow diagrams without implying a semantic IDEF5 relation. These are connector tools: drag from the toolbox, then click source then target.

Connect (Forward)
Directed flow from source to target. A forward-pointing chevron (▶) at the midpoint indicates direction.
Connect (Backward)
Directed flow from target back to source. A backward-pointing chevron (◀) at the midpoint.
Connect (Plain)
An undirected connection with no implied direction. Used to associate elements, for example linking a process node to the circle of a state transition node.

Grouping & Annotation

Container

A dashed rectangle for visually grouping related symbols. Drag it from the toolbox, place it on the canvas, and resize it to enclose the nodes you want to group. Containers are purely organisational and have no effect on connections or semantics. The label appears in the top-left corner.

Referent

A classification schema node used to identify kinds with a structured labelling scheme. It has three fields editable in the Properties panel:

  • Reference ID: a short identifier (e.g. a code or number), shown in the top-left cell
  • Method Name: the naming convention or classification method, shown in the top-right cell
  • Label: the human-readable concept name, shown in the bottom half

Key

A legend box that explains the symbols used in the diagram. Add entries in the Properties panel by choosing a symbol type and typing a label. Include a key whenever your diagram will be read by someone unfamiliar with IDEF5 notation.

Save, load & export

ButtonAction
New Clear the canvas and start a new diagram. You will be asked to confirm if there is existing content on the canvas.
Save Download the current diagram as a .json file. This file captures everything (nodes, edges, labels, positions) and can be loaded back into Quiddity at any time.
Load Open a previously saved .json file. The current diagram is replaced by the loaded one.
SVG Export the current diagram as a scalable vector image (.svg). SVG files can be opened in a web browser, edited in tools like Inkscape, or embedded in Word and PowerPoint documents.
Caution: Quiddity auto-saves to your browser's local storage, but this is not a substitute for a proper save. Browser data can be cleared. Always use Save to keep a permanent copy of important work.

Built-in examples

Click Examples ▾ in the toolbar to load any of four pre-built diagrams:

ExampleWhat it demonstrates

Water Phase Transitions State transition nodes (weak and strong), showing how kinds change state under conditions.

Warning: Loading an example replaces the current diagram. Save your work first if needed.

Auto-save

Quiddity automatically saves the current diagram to your browser's local storage on every change. When you reopen the file in the same browser, your work is restored from local storage.

Auto-save is separate from the Save button. Local storage is scoped to the browser and the file path. Opening Quiddity from a different location, a different browser, or after clearing browser data will result in an empty diagram.

Using on iPad / iPhone

Quiddity works on iOS and iPadOS in Safari. Touch gestures replace mouse and keyboard interactions throughout.

Navigating the canvas

ActionTouch gesture
PanDrag with one finger on empty canvas space
Zoom in / outPinch with two fingers, or use the + / − toolbar buttons
Fit to windowTap Fit in the toolbar

Adding nodes

Touch and hold a symbol in the toolbox, then drag it onto the canvas. A translucent copy of the symbol follows your finger. Lift your finger over the canvas to place the node.

Drawing connections

  1. Touch and drag a connector item (e.g. State Trans. (Strong), Connect (Plain)) from the toolbox and drop it onto the source node on the canvas.
  2. Tap the target node to complete the connection.

Tap Esc is not available on a touch keyboard. Tap any empty canvas area and then start again, or use the on-screen keyboard's Escape key if your keyboard has one.

Selecting and moving nodes

ActionTouch gesture
Select a nodeTap it
Move a nodeTap to select, then drag
Multi-selectNot available without a keyboard. Select and act on one node at a time.
DeselectTap empty canvas space

Editing labels

Double-tap any node to open the inline label editor. The on-screen keyboard will appear. Tap outside the editor or press Return to confirm.

Context menu

The context menu is not available on iOS. It requires a right-click. To delete a selected node or edge, select it and use the Delete key on a connected hardware keyboard. Duplicate and layer-ordering options are not currently accessible without a mouse.

Toolbar buttons

All toolbar buttons (New, Save, Load, SVG, Undo, Redo, Fit, Examples) work normally with a tap. Save and SVG will trigger the iOS share sheet, from which you can save the file to Files, send it by email, or copy it.

Tip: For the best experience on iPad, add Quiddity to your Home Screen via the Share button in Safari. This opens it full-screen without the browser chrome.
Note: Keyboard shortcuts (Ctrl+Z, Delete, etc.) require a hardware keyboard connected to the iPad. Without a keyboard, use the toolbar Undo/Redo buttons and the long-press context menu to delete elements.

About IDEF5

IDEF5 (Integrated Definition for Ontology Description Capture) is a formal notation for modelling ontologies: systematic descriptions of the kinds of things that exist in a domain, the properties they have, and the relationships between them.

An IDEF5 schematic answers questions like:

  • What kinds of things exist in this domain?
  • What are the essential properties of each kind?
  • How are kinds related to each other (hierarchy, composition, association)?
  • How do things in this domain change state over time?
  • What processes act on things in this domain?

Core concepts

ConceptWhat it means
Kind A natural category (not just a set, but a type with essential defining properties shared by all its instances), e.g. "Water", "Metal", "Vehicle".
Individual A specific named thing: a particular instance of a kind (e.g. "The Pacific Ocean", "Bolt #42").
Relation An association between kinds or individuals that holds as a matter of domain fact (e.g. "contains", "is made of").
Subkind-of Taxonomic specialisation: every instance of the subkind is necessarily an instance of the superkind (e.g. "Bolt subkind-of Fastener").
Part-of Mereological composition: one kind or individual is a constituent of another (e.g. "Ink subkind-of Ballpoint Pen component").
State transition A change in the state of a kind or individual. Weak transitions are possible; strong transitions are necessary and irreversible.
